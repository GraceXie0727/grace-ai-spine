# grace-no-yesman：不让 AI 顺着你

一个 Claude Code 的 Agent Skill，把 AI 的讨好模式关掉，专门给你的**方案**挑逻辑毛病。

你有一个方案、计划、决策，或者一个产品/内容的想法——发给它，它不会说"这个方向很好，可以考虑优化一下"那种废话，而是逐条引用你方案里的原话举证，用一套逻辑武器把漏洞挖出来，最后直接裁决：**可推进 / 得先改这几处 / 建议推翻重来**。

解决的问题：**AI 默认只会鼓掌。** 你让它看方案，它先夸三段再轻轻带一句"也许可以再想想"。这个 skill 就是让它闭嘴鼓掌、开口讲逻辑。

---

## 它跟一般"帮我看看"哪里不一样

严格只有一个来源：**逻辑**。不是靠语气凶，是靠"这一步到底站不站得住"。它审方案时随身带五件逻辑武器：

| 逻辑武器 | 干什么 | 来自 |
|---|---|---|
| 能不能被证伪 | 逼问"这方案什么情况下算失败"，说不出就是信念不是方案 | 波普尔 |
| 凭什么这么干就能成 | 拆"主张 → 依据 → 中间那座桥"，桥断了就是逻辑跳跃 | 图尔敏 |
| 连环追问隐藏前提 | 一路问"为什么"，逼出没说出口的默认假设 | 苏格拉底 |
| 要搞垮它最快的办法 | 反过来想：想让它失败该从哪下手、最先崩哪块 | 芒格 |
| 逻辑谬误对照 | 抓以偏概全、虚假两难、幸存者偏差、循环论证…… | 经典谬误清单 |

报告里全说大白话，名人只在招式边点一句，不掉书袋。

## 安装

前提：你在用 [Claude Code](https://claude.com/claude-code) 或其他支持 Agent Skills 的工具。

### 方式一：一行命令安装（推荐）

```
npx skills add https://github.com/GraceXie0727/grace-no-yesman --skill grace-no-yesman
```

### 方式二：把下面这段话直接发给 AI

> 帮我安装 `grace-no-yesman` 这个 Claude Code skill。请：
> 1. 确保 `~/.claude/skills/` 目录存在（不存在就创建）
> 2. 执行 `git clone https://github.com/GraceXie0727/grace-no-yesman.git` 到临时目录
> 3. 把临时目录里的 `grace-no-yesman/` 文件夹复制到 `~/.claude/skills/`
> 4. 验证：`ls ~/.claude/skills/grace-no-yesman/` 应该能看到 `SKILL.md`
> 5. 告诉我装好了，之后我说"审审这个方案"就会触发它

把这段复制粘贴给 Claude Code / Cursor / 任何有 shell 权限的 AI Agent，它会自动装好。

### 方式三：手动

点本页右上角绿色的「Code」按钮 →「Download ZIP」，解压后把 `grace-no-yesman` 文件夹（里面有 `SKILL.md`）复制到 `~/.claude/skills/`（Windows 是 `C:\Users\你的用户名\.claude\skills\`），重启 Claude Code。

装完后目录长这样：

```
~/.claude/skills/
└── grace-no-yesman/
    └── SKILL.md
```

## 开始使用

把你想审的方案（计划、决策、产品或内容想法）发给 Claude Code，然后说「审审这个方案」或 `/grace-no-yesman`。

它会输出一份这样的报告：

```
方案审查

结论：建议推翻重来——"火 → 有人看 → 过万 → 变现"整条链没有一处逻辑站得住
最大的问题：把"日更三个月"当成了"粉丝过万"的充分条件，中间没有任何因果

具体问题（按重要程度排）：

1. 把"勤奋"当成了"结果"
   哪里不对：方案说「只要坚持日更三个月，粉丝就能过万」，日更和涨粉之间没有因果桥
   怎么改：换成一个可验证的涨粉假设，拿同类号真实数据校准

2. "赛道火"推不出"有你的位置"
   哪里不对：方案说「现在 AI 这么火……肯定很多人看」，火恰恰意味着竞争最凶
   怎么改：把"AI 火"换成"哪个细分人群、哪个没被满足的具体需求"

还差什么才能判准：没说清差异化——凭什么读者看你的、不看别人的
```

## 设计原则

- **没有证据不许开口。** 每条批评必须引用你方案里的原话 + 说清违反了哪条逻辑。"感觉不太行"这种话禁止。
- **用逻辑，不用感觉。** 每条批评都挂在一件逻辑武器上，不是凭态度凶。
- **只挑不改。** 它指出问题和方向，但不替你重写方案——改是你自己的事。
- **不为凶而凶。** 方案真的扎实就明说无致命伤，但照样把最弱一环挖给你看。
